# Data Directory

## Real Dataset (Required for Full Pipeline)

**Dataset:** Kaggle Healthcare Provider Fraud Detection Analysis
**URL:** https://www.kaggle.com/datasets/rohitrox/healthcare-provider-fraud-detection-analysis

### Manual Download Steps:
1. Go to https://www.kaggle.com/datasets/rohitrox/healthcare-provider-fraud-detection-analysis
2. Click "Download" (requires free Kaggle account)
3. Unzip the downloaded file
4. Place all CSV files in `data/raw/`

### Expected Files:
- `Train_Beneficiarydata-*.csv` — beneficiary demographics (age, chronic conditions, etc.)
- `Train_Inpatientdata-*.csv` — inpatient claims (admission dates, reimbursement, physicians)
- `Train_Outpatientdata-*.csv` — outpatient claims
- `Train-*.csv` — provider fraud labels (Provider, PotentialFraud: Yes/No)
- `Test_Beneficiarydata-*.csv` (optional)
- `Test_Inpatientdata-*.csv` (optional)
- `Test_Outpatientdata-*.csv` (optional)
- `Test-*.csv` (optional)

The pipeline auto-detects files by keyword in filename (case-insensitive), so exact
timestamp suffixes don't matter.

## Synthetic Demo Data (Available Without Download)

The file `data/raw/synthetic_claims.csv` is generated by running:

```bash
python src/synthetic_data_generation.py
```

This synthetic dataset demonstrates the FWA analytics workflow without requiring
Kaggle credentials. It generates 5,000 synthetic insurance claims with a hidden
latent fraud-intent variable to produce realistic (non-trivial) model metrics.

## Data Disclaimer

- This project does NOT contain real patient data, clinical records, or insurance company data.
- The Kaggle dataset is a public educational resource for learning fraud detection techniques.
- NOT Manulife data. NOT John Hancock data. NOT Long Term Care-specific data.
- No PHI (Protected Health Information) is present anywhere in this repository.
- RAG policy text and claim documents are fully synthetic, written for demonstration purposes.
